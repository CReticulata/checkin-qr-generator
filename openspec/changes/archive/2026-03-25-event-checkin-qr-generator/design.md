## 背景說明

這是一個全新的 Next.js 活動報到管理應用程式。系統需要透過參加者已註冊的 Google 帳號進行身份驗證，並產生獨特的 QR Code 供活動入場驗證使用。此應用程式將由活動參加者在抵達場地前，透過行動裝置或電腦使用。

**現況：** 無現有系統。從頭開始使用 Next.js 框架建立。

**限制條件：**

- CSV 檔案格式固定：電子郵件位於第 5 欄（索引 4），票券編號位於第 2 欄（索引 1）
- 必須使用 Google OAuth（參加者使用 Google 帳號註冊）
- QR Code 格式必須為 `EVENT_ID:票券後七碼`（例如：票券 GOOGA260000000 產生 111111:0000000）
- CSV 檔案包含敏感的參加者資料，不得公開存取

**利害關係人：**

- 活動主辦單位（管理員）
- 活動參加者（終端使用者）

## 目標 / 非目標

**目標：**

- 提供無縫的 Google OAuth 身份驗證
- 以 O(1) 效能高效解析 CSV 資料進行電子郵件查詢
- 產生包含 EVENT_ID:票券後七碼的可掃描 QR Code
- 妥善處理邊界情況（未註冊電子郵件、缺少資料）
- 確保 CSV 資料安全（無法透過公開 URL 存取）
- 透過可設定的 EVENT_ID 支援多個活動

**非目標：**

- 管理介面供上傳 CSV（可接受手動放置檔案）
- 即時報名資料更新（CSV 為靜態）
- QR Code 掃描功能（僅生成）
- 多語言支援
- 行動應用程式（僅限網頁版）
- 資料庫儲存（CSV 檔案為唯一真實來源）

## 技術決策

### 1. 身份驗證：NextAuth.js 搭配 Google Provider

**決策：** 使用 NextAuth.js 函式庫搭配 Google OAuth provider。

**理由：**

- NextAuth.js 是 Next.js 的標準身份驗證解決方案，內建 OAuth 支援
- 提供會話管理、回呼函式與安全最佳實踐
- Google OAuth 符合基於電子郵件的身份驗證需求
- 比建立自訂 OAuth 流程更簡單

**考慮的替代方案：**

- 自訂 OAuth 實作 → 拒絕：更複雜、容易出錯且難以維護
- Clerk/Auth0 → 拒絕：增加外部相依性與成本；NextAuth.js 已足夠

### 2. CSV 處理：記憶體內 Map 搭配 csv-parse

**決策：** 在應用程式啟動時將 CSV 解析為記憶體內的 `Map<string, string>` 結構。

**理由：**

- 符合要求的 O(1) 查詢效能
- csv-parse 正確處理邊界情況（引號欄位、內嵌逗號）
- 記憶體內儲存對唯讀資料來說快速且簡單
- 檔案在啟動時解析一次，減少 I/O 負擔

**實作細節：**

- 在 `src/lib/csvLoader.ts` 中將 Map 儲存為模組層級變數
- 將電子郵件標準化為小寫以進行不區分大小寫的比對
- 跳過標頭列（索引 0）
- 匯出非同步函式 `getTicketNumber(email: string): Promise<string | null>`

**考慮的替代方案：**

- SQLite 資料庫 → 拒絕：為簡單的唯讀查詢增加複雜度
- 每次請求讀取 CSV → 拒絕：效能不佳、不必要的 I/O

### 3. CSV 檔案位置：src/data/ 目錄

**決策：** 將 `participants.csv` 放置在 `src/data/` 目錄，而非 `public/`。

**理由：**

- `src/` 中的檔案會隨應用程式打包，不會作為靜態資源公開
- 防止直接透過 URL 存取敏感的參加者資料
- 可在伺服器元件或 API 路由中透過 Node.js `fs` 模組讀取

**安全注意事項：** 如果 CSV 在開發期間包含真實資料，請將 `src/data/` 加入 `.gitignore`。

### 4. QR Code 生成：伺服器端使用 qrcode 函式庫

**決策：** 使用 `qrcode` npm 套件在伺服器端生成 QR Code，回傳資料 URL 給客戶端。

**理由：**

- 成熟、維護良好的函式庫，具有簡單的 API
- 可產生資料 URL（base64 編碼影像）供 React 輕鬆顯示
- 伺服器端生成可將敏感邏輯保留在伺服器端
- 與 Next.js 伺服器元件或 API 路由配合良好

**QR Code 格式處理：**

- 自動擷取票券編號的後七碼
- 格式：`EVENT_ID:票券後七碼`
- 範例：票券 GOOGA260000000 → QR Code 內容 111111:0000000

**考慮的替代方案：**

- 客戶端生成（qrcode.react）→ 拒絕：將票券查詢邏輯暴露給客戶端
- SVG 生成 → 拒絕：對此使用情境來說，資料 URL 影像更簡單

### 5. 架構：伺服器元件搭配身份驗證 API 路由

**決策：**

- 使用 Next.js 13+ App Router 搭配伺服器元件
- 儀表板頁面（`src/app/dashboard/page.tsx`）作為伺服器元件
- 透過 NextAuth.js API 路由進行身份驗證（`src/app/api/auth/[...nextauth]/route.ts`）

**理由：**

- 伺服器元件可直接讀取 CSV 資料而不暴露給客戶端
- 將敏感操作（CSV 查詢、QR Code 生成）保留在伺服器端
- NextAuth.js 需要 API 路由處理 OAuth 回呼
- 比獨立的 API + 客戶端架構更簡單

**資料流程：**

1. 使用者點擊「使用 Google 登入」→ 重新導向至 Google OAuth
2. Google 重新導向回 NextAuth.js 回呼
3. NextAuth.js 使用使用者電子郵件建立會話
4. 儀表板頁面（伺服器元件）讀取會話，從 CSV 查詢票券編號
5. 若找到，生成 QR Code 並渲染；若未找到，重新導向至錯誤頁面

### 6. 錯誤處理：專用錯誤頁面

**決策：** 建立 `/error` 頁面，為未註冊電子郵件顯示「找不到報名資料」訊息。

**理由：**

- 清楚區分成功與錯誤狀態
- 比儀表板上的內嵌錯誤訊息有更好的使用者體驗
- 便於為不同情境自訂錯誤訊息

## 風險 / 權衡

### 1. 記憶體內 CSV 儲存

**風險：** 對於非常大的 CSV 檔案（>100k 參加者）可能造成高記憶體使用量。
**緩解措施：** Map 結構記憶體效率高（僅儲存電子郵件與票券編號）。若需要大規模活動，可視需求遷移至資料庫。

### 2. 靜態 CSV 資料

**風險：** CSV 更新需要重新啟動應用程式以重新載入資料。
**緩解措施：** 記錄更新 CSV 後需要重新啟動應用程式的必要性。若稍後需要即時更新，可考慮檔案監視機制。

### 3. 單一活動 ID

**風險：** 執行多個同時進行的活動需要獨立部署或變更設定。
**緩解措施：** 使用環境變數設定 EVENT_ID。未來增強功能可支援多租戶。

### 4. CSV 欄位索引相依性

**風險：** 若 CSV 格式變更（欄位重新排序），應用程式會失效。
**緩解措施：** 在啟動時驗證 CSV 結構。若預期欄位缺少或錯位，記錄清楚的錯誤訊息。

### 5. 電子郵件大小寫敏感性

**風險：** 電子郵件大小寫不符導致查詢失敗（使用者以「User@Example.com」登入但 CSV 有「user@example.com」）。
**緩解措施：** 在 CSV 解析與查詢期間將所有電子郵件標準化為小寫。

### 6. Google OAuth Token 過期

**風險：** 使用者會話在活動期間過期，需要重新登入。
**緩解措施：** 設定 NextAuth.js 會話過期時間（預設 30 天）。若需要，使用者可重新登入。

### 7. QR Code 票券編號擷取

**風險：** 若票券編號長度少於 7 碼，可能造成非預期結果。
**緩解措施：** `slice(-7)` 對短於 7 碼的字串仍能正常運作（回傳完整字串）。若需要，可加入長度驗證。

## 部署計畫

### 初始部署

1. 設定 Google Cloud Console 專案並取得 OAuth 憑證
2. 使用 `GOOGLE_CLIENT_ID`、`GOOGLE_CLIENT_SECRET`、`NEXTAUTH_SECRET`、`EVENT_ID` 設定 `.env.local`
3. 將 `participants.csv` 放置在 `src/data/` 目錄
4. 執行 `npm install` 安裝相依套件
5. 建置並部署 Next.js 應用程式（`npm run build && npm start` 或部署至 Vercel）
6. 使用已註冊與未註冊電子郵件進行測試

### 回復策略

- 由於這是新應用程式，回復意味著移除部署
- 不需要資料遷移（CSV 為唯一真實來源）
- 若發生問題，可在修正後快速重新部署

### 部署後驗證

- 驗證 Google OAuth 登入流程運作正常
- 為樣本已註冊電子郵件測試 QR Code 生成
- 確認未註冊電子郵件顯示錯誤頁面
- 檢查無法透過公開 URL 存取 CSV 檔案（嘗試存取 `/data/participants.csv`）
- 驗證 QR Code 格式正確（EVENT_ID:票券後七碼）
